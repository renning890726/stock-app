# 产品需求文档 (PRD)
## 勇闯大A - AI驱动智能投资建议系统

**版本**: v1.0  
**文档日期**: 2026年2月13日  
**项目类型**: 全栈Web应用  
**目标用户**: A股投资者、个人投资者、中小投资者

---

## 一、产品概述

### 1.1 产品简介
"勇闯大A"是一个基于AI大语言模型和实时数据驱动的A股智能投资助手系统，旨在为个人投资者提供持仓管理、实时监控、智能分析和消息推送等一站式投资辅助服务。

### 1.2 产品定位
- **目标市场**: A股个人投资者、中小投资者
- **核心价值**: 降低投资门槛，提升投资决策质量，实现智能化的投资组合管理
- **差异化优势**: 
  - AI驱动的大语言模型分析
  - 实时股价监控与异动检测
  - 智能推送机制（飞书机器人）
  - 多维度投资分析报告

### 1.3 产品愿景
成为A股个人投资者的智能投资顾问，帮助投资者实现：
- 科学管理投资组合
- 及时捕捉市场机会
- 降低投资风险
- 提升投资收益

## 二、用户角色与场景

### 2.1 用户角色

| 角色 | 描述 | 主要需求 |
|------|------|----------|
| **个人投资者** | 有一定A股投资经验的个人用户 | 持仓管理、行情监控、投资建议、风险预警 |
| **中小投资者** | 资金规模较小的投资者 | 低成本、易使用的投资辅助工具 |
| **长期投资者** | 偏好中长期投资的用户 | 持仓分析、定期复盘、趋势判断 |

### 2.2 使用场景

#### 场景1: 持仓管理
**用户故事**: 作为一名投资者，我希望能够方便地管理我的股票持仓，包括添加、修改、删除持仓信息，并实时查看持仓盈亏情况。

**前置条件**: 已登录系统  
**主要流程**:
1. 进入"持仓管理"页面
2. 查看当前持仓列表
3. 点击"添加持仓"按钮，输入股票代码、名称、持有数量、成本价
4. 保存持仓信息
5. 系统自动获取实时股价并计算盈亏

**后置条件**: 持仓信息已保存，实时数据已更新

#### 场景2: 实时监控与异动提醒
**用户故事**: 作为一名投资者，我希望系统能够实时监控我的持仓股票，当股票出现异动时及时通知我。

**前置条件**: 已添加持仓，已配置飞书Webhook  
**触发条件**: 
- 单只股票当日涨跌幅超过5%
- 创3个月历史新高
- 每周五下午定期复盘

**主要流程**:
1. 系统每分钟监控一次持仓股票
2. 检测到异动（涨跌幅>5%或创历史新高）
3. 生成投资建议（买入/卖出/持有）
4. 通过飞书机器人推送消息
5. 用户收到消息并做出投资决策

#### 场景3: 智能分析报告
**用户故事**: 作为一名投资者，我希望能够获得基于AI生成的深度投资分析报告，帮助我更好地理解市场趋势和个股情况。

**前置条件**: 已添加持仓  
**主要流程**:
1. 进入"投资建议"页面
2. 点击"生成持仓分析报告"
3. 系统收集所有持仓股票的数据
4. AI生成多维度分析报告
5. 展示报告内容，包括：
   - 整体持仓概况
   - 个股深度分析
   - 操作建议
   - 风险提示

#### 场景4: 目标设置
**用户故事**: 作为一名投资者，我希望能够设置我的年度投资目标和投资风格，系统根据我的目标提供个性化的建议。

**前置条件**: 已登录系统  
**主要流程**:
1. 进入"目标设置"页面
2. 输入持仓总金额
3. 设置年度盈利目标（如20%）
4. 选择投资风格（短线/中线/长线）
5. 选择AI分析模型
6. 配置飞书Webhook（可选）
7. 保存配置

## 三、功能需求

### 3.1 核心功能模块

#### 3.1.1 目标设置模块
**功能描述**: 允许用户设置投资目标、投资风格和系统配置

**详细需求**:

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 持仓金额设置 | 设置总投资金额，用于计算盈利目标 | P0 |
| 盈利目标设置 | 设置年度盈利目标百分比（默认20%） | P0 |
| 投资风格选择 | 选择短线、中线、长线等投资风格 | P1 |
| LLM模型选择 | 选择AI分析模型（豆包/DeepSeek/Kimi等） | P1 |
| 飞书Webhook配置 | 配置飞书机器人Webhook地址 | P2 |
| 现金管理 | 记录可用现金金额 | P2 |

**数据字段**:
- `positionAmount`: 持仓总金额（数值）
- `profitTarget`: 年度盈利目标（百分比）
- `tradingStyle`: 投资风格（short_term/medium_term/long_term）
- `llmModel`: LLM模型选择
- `feishuWebhookUrl`: 飞书Webhook地址
- `cash`: 可用现金金额

#### 3.1.2 持仓管理模块
**功能描述**: 管理用户的股票持仓信息，提供实时价格和盈亏计算

**详细需求**:

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 添加持仓 | 添加新的股票持仓 | P0 |
| 编辑持仓 | 修改持仓信息（数量、成本价） | P0 |
| 删除持仓 | 删除持仓记录 | P0 |
| 内联编辑 | 支持直接在表格中修改持股数量 | P1 |
| 手动编辑价格 | 支持手动输入正确的股价 | P1 |
| 刷新价格 | 手动刷新所有持仓的实时价格 | P0 |
| 股票分析 | 查看单只股票的深度分析 | P1 |
| 价格来源标识 | 显示价格来源（akshare/sina/search等） | P2 |

**数据字段**:
- `id`: 持仓记录ID
- `stockName`: 股票名称
- `stockCode`: 股票代码
- `quantity`: 持股数量
- `costPrice`: 成本价
- `currentPrice`: 当前价格（实时）
- `priceSource`: 价格来源
- `openPrice`: 开盘价
- `changePercent`: 涨跌幅
- `peRatio`: 市盈率

**实时价格获取策略**:
1. 优先使用 Python Akshare 服务（端口9001）
2. 降级到新浪财经API
3. 降级到东方财富API
4. 最后降级到Search API

**缓存策略**:
- 交易时间（9:30-11:30, 13:00-15:00）：缓存30秒
- 非交易时间：缓存24小时
- 支持手动清除缓存

#### 3.1.3 投资建议模块
**功能描述**: 基于AI大语言模型生成智能投资建议和分析报告

**详细需求**:

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 生成持仓分析报告 | 基于所有持仓生成综合分析报告 | P0 |
| 生成个股分析报告 | 对单只股票进行深度分析 | P0 |
| 实时监控建议 | 基于异动生成即时操作建议 | P0 |
| 热点推送 | 推送市场热点分析 | P1 |
| 历史建议查看 | 查看历史投资建议记录 | P2 |
| 模型对比 | 对比不同AI模型的建议差异 | P2 |

**分析报告内容**:
- **整体持仓概况**
  - 持仓总市值
  - 整体盈亏情况
  - 盈利目标达成率
  - 风险评估
  
- **个股深度分析**
  - 基本信息（代码、名称、行业）
  - 价格走势（开盘、收盘、最高、最低）
  - 技术指标（MACD、KDJ、RSI、布林带）
  - 资金流向
  - 新闻研报（7天内）
  
- **操作建议**
  - 买入/卖出/持有
  - 具体操作理由
  - 风险提示
  - 参考价格区间

**多轮搜索架构**:
- **第一轮**: 市场概况（整体走势、指数表现、市场情绪、热点板块）
- **第二轮**: 个股详情（最新消息、技术面分析）
- **第三轮**: 深度技术分析（支撑位、阻力位、K线形态、机构研报目标价）

#### 3.1.4 实时监控模块
**功能描述**: 实时监控持仓股票，检测异动并触发推送

**详细需求**:

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 异动检测 | 检测当日涨跌幅超过5%的股票 | P0 |
| 历史新高检测 | 检测创3个月历史新高的股票 | P0 |
| 定期复盘 | 每周五下午定期推送复盘报告 | P1 |
| 监控状态查看 | 查看监控任务运行状态 | P2 |
| 手动触发监控 | 手动触发一次监控任务 | P2 |

**触发条件**:
1. **异动触发**: 当日涨跌幅 ≥ 5%
2. **历史新高**: 当前价格 > 3个月历史最高价
3. **定期推送**: 每周五下午14:30

**推送内容**:
- 异动股票列表及涨跌幅
- 历史新高股票列表
- AI生成的操作建议
- 触发原因说明

**防重复推送机制**:
- 同一股票每天只推送一次
- 使用 `lastAlertDate` 字段记录上次推送时间
- 对比日期判断是否已推送

#### 3.1.5 系统配置模块
**功能描述**: 系统级别的配置和管理功能

**详细需求**:

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 价格缓存管理 | 查看和清除价格缓存 | P2 |
| 监控任务管理 | 查看和测试监控任务 | P2 |
| 调试工具 | 用于调试和问题排查 | P3 |
| 日志查看 | 查看系统运行日志 | P3 |

### 3.2 非功能性需求

#### 3.2.1 性能需求
- **响应时间**: 
  - 页面加载 < 2秒
  - API响应 < 1秒
  - 实时价格更新 < 30秒
  
- **并发能力**: 支持100+并发用户

#### 3.2.2 可靠性需求
- **可用性**: 99.5%以上
- **数据一致性**: 持仓数据强一致性，价格数据最终一致性
- **容错能力**: 单个API失败不影响整体功能

#### 3.2.3 安全性需求
- **数据加密**: 数据库连接使用SSL加密
- **访问控制**: API接口权限控制
- **数据备份**: 定期备份用户配置和持仓数据

#### 3.2.4 可扩展性需求
- **水平扩展**: 支持多实例部署
- **模块化设计**: 各模块独立，便于扩展
- **插件化**: 支持添加新的数据源和分析模型

- 统一返回JSON格式
- 使用标准HTTP状态码

#### API响应格式
```json
{
  "success": true,
  "data": {},
  "error": null,
  "timestamp": 1770959400000
}
```
